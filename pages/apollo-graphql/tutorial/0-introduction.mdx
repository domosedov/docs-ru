Добро пожаловать! Это руководство научит вас создавать fullstack-приложения на базе GraphQL с платформой Apollo.

Мы хотим, чтобы вы чувствовали себя настоящим специалистом, умеющим создавать "production-ready" приложения с Apollo, поэтому мы пропускаем «Hello World» в пользу примера, который "ближе к рельности", с аутентификацией, пагинацией, тестированием и многим другим.

Готовы? Тогда начинаем!

## Что мы будем создавать

В этом руководстве, мы будем создавать интерактивное приложение для резервирования места на предстоящий запуск SpaceX. Думайте об этом как об Airbnb для космических путешествий! Все данные реальны благодаря [SpaceX-API](https://github.com/r-spacex/SpaceX-API).

Вот как будет выглядеть готовое приложение:

<Image src="/images/space-explorer.png"/>

Приложение включает в себя следующие страницы-представления(Views):

* Логин
* Список предстоящих запусков
* Детальные данные для индивидуального запуска
* Профиль пользователя
* Корзина

Чтобы заполнить эти представления данными, граф нашего приложения будет подключаться к двум источникам: REST API и базе данных SQLite. (Не волнуйтесь, вам не нужно быть знакомым ни с одной из этих технологий, чтобы пройти обучение.)

Как уже упоминалось ранее, мы хотим, чтобы этот пример напоминал реальное Apollo-приложение, поэтому мы также добавим общие полезные функции, такие как аутентификация, пагинация и управление состоянием.

## Предварительные условия

В этом руководстве предполагается, что вы знакомы как с JavaScript / ES6, так и с React. Если вам нужно освежить в памяти React, мы рекомендуем пройти [официальное руководство](https://ru.reactjs.org/docs/getting-started.html).

> Создание интерфейса с помощью React не является обязательным требованием для использования платформы Apollo, но он основной, поддерживаемый Apollo. Если вы используете другой фреймворк для создания интерфейса (например, Angular или Vue), вы все равно можете применить к нему концепции этого руководства.

### Системные требования

Прежде чем мы начнем, убедитесь, что у вас установлено следующее:

- [Node.js](https://nodejs.org/) v8.x или новее
- [npm](https://www.npmjs.com/) v6.x или новее
- [git](https://git-scm.com/) v2.14.1 или новее

Хотя это не обязательно, мы также рекомендуем использовать [VS Code](https://code.visualstudio.com/)
в качестве редактора, чтобы вы могли использовать полезное [расширение](https://marketplace.visualstudio.com/items?itemName=apollographql.vscode-apollo).

## Клонирование примера приложения

Теперь начинается самое интересное! Из предпочитаемой вами директории клонируйте [этот репозиторий](https://github.com/apollographql/fullstack-tutorial):

```bash
git clone https://github.com/apollographql/fullstack-tutorial.git
```

Репозиторий содержит две папки: start и final. Во время обучения вы будете редактировать файлы в папке start, и в конце они будут соответствовать завершенному приложению в папке final.

Каждая папка содержит еще две: `server` и `client`. Сначала мы будем работать с папкой server. Если вы уже комфортно себя чувствуете в создании graph API, то вы можете пропустить этот шаг и перейти к создания клиентской части во [второй половине руководства](/apollo-graphql/tutorial/6-client).
